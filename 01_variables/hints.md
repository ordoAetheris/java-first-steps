# Подсказки: Переменные и типы данных

## Задание 1: Визитка
- `System.out.println("Имя: " + name);` — конкатенация строк через `+`

## Задание 2: Обменник
- Целая часть: `(int)(rubli / kurs)` — приведение к int отбрасывает дробную часть
- Остаток: `rubli - dollars * kurs` или через `%` (но `%` для double работает иначе)

## Задание 3: Размер файла
- GB: `bytes / (1024 * 1024 * 1024)`
- Остаток после GB: `bytes % (1024 * 1024 * 1024)`
- Осторожно: `1024 * 1024 * 1024` может переполнить int! Используйте `1024L * 1024 * 1024`

## Задание 4: Время
- Общее время в минутах: `(int)(distance / speed * 60)`
- Часы: `totalMinutes / 60`, минуты: `totalMinutes % 60`

## Задание 5: Целочисленная ловушка
- Когда ОБА операнда int → результат int. `10 / 3 = 3` (дробная часть отброшена)
- Когда ХОТЯ БЫ ОДИН double → результат double. `10.0 / 3 = 3.333...`
- `1 / 2 = 0` — самая частая ошибка новичков!

## Задание 6: Переполнение
- int хранит 32 бита. MAX_VALUE = 01111...1 (31 единица). +1 → 10000...0 = MIN_VALUE
- Это как одометр: 999999 + 1 = 000000
- Java НЕ бросает ошибку при переполнении! Просто тихо "обнуляется"
- Если нужны большие числа — `long` или `BigInteger`

## Задание 7: Точность double
- 0.1 в двоичной системе = бесконечная дробь (как 1/3 в десятичной)
- Компьютер хранит приближение, не точное значение
- Сравнивать: `Math.abs(a - b) < epsilon` где epsilon = 0.0001 (или меньше)
- Для денег: НИКОГДА не используйте double! Используйте `BigDecimal` или `long` в копейках

## Задание 8: Приведение типов
- `(int) long`: если значение > MAX_VALUE → обрезание (как задание 6)
- `(int) double`: отбрасывание дробной части (не округление!)
- `char → int`: символ = число (ASCII/Unicode код). 'A' = 65, 'B' = 66
- `int → char`: число → символ

## Задание 9: Аргументы
- `args` — массив строк. `args.length` = количество аргументов
- `args[0]` — первый аргумент (не имя программы, как в C!)
- `Integer.parseInt("28")` — строка → число
- Без аргументов: `ArrayIndexOutOfBoundsException` если не проверить `args.length`

## Задание 10: Кофейня
- `System.out.printf` — форматированный вывод
- `"%-20s"` — строка, выровненная влево, 20 символов
- `"%8.2f"` — число, 8 символов ширина, 2 знака после точки
- Порядок: подитог → скидка → подитог-скидка → НДС от (подитог-скидка) → итого
