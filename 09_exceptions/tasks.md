# Задания: Исключения (Exceptions — обработка ошибок)

## Задание 1: Деление с обработкой ★☆☆
Файл `Task01.java`. Два числа из аргументов. Делим первое на второе. Если второе = 0 → поймать ArithmeticException (ошибка арифметики).
```bash
java Task01 10 0    # Ошибка: деление на ноль
java Task01 10 3    # Результат: 3.33
```

## Задание 2: Парсинг числа из строки ★☆☆
Файл `Task02.java`. Аргумент — строка. Преобразовать в int. Если не число → поймать NumberFormatException (ошибка формата числа).
```bash
java Task02 42      # Число: 42
java Task02 abc     # Ошибка: 'abc' — не число
java Task02 3.14    # Ошибка: '3.14' — не целое число
```

## Задание 3: Свой exception — InsufficientFundsException ★★☆
Файлы: `InsufficientFundsException.java`, `BankAccount.java`, `Task03.java`. При withdraw (снятие) больше баланса → бросить (throw) своё исключение с деталями.
```bash
java Task03
# Баланс: 1000. Снимаем 500 → OK. Баланс: 500.
# Баланс: 500. Снимаем 1500 → Ошибка: Недостаточно средств (баланс: 500, запрошено: 1500)
```

## Задание 4: Множественный catch (несколько типов ошибок) ★★☆
Файл `Task04.java`. Аргумент — индекс массива. Может бросить ArrayIndexOutOfBoundsException (выход за границы) ИЛИ NumberFormatException (не число). Поймать оба.
```bash
java Task04 2       # Элемент [2]: яблоко
java Task04 99      # Ошибка: индекс 99 за пределами массива (размер: 5)
java Task04 abc     # Ошибка: 'abc' — не число
```

## Задание 5: finally — гарантированное закрытие ★★☆
Файл `Task05.java`. Открыть файл, прочитать, ЗАКРЫТЬ даже при ошибке. Два варианта: finally и try-with-resources (автоматическое закрытие). Сравнить.

## Задание 6: Стек вызовов (stack trace) ★★★
Файл `Task06.java`. Метод A() вызывает B(), B() вызывает C(), C() бросает exception. Поймать в A(), вывести полный стек вызовов: видно КТО вызвал КОГО.

## Задание 7: Валидатор формы — собрать ВСЕ ошибки ★★★
Файл `Task07.java`. Поля: имя, email, возраст. Проверить ВСЕ и собрать ошибки в список (не останавливаться на первой).
```bash
java Task07 "" "not-email" "-5"
# Ошибки:
# - Имя не может быть пустым
# - Email должен содержать @
# - Возраст не может быть отрицательным
```

## Задание 8: throws — пробросить вызывающему ★★☆
Файл `Task08.java`. Метод `readConfig(String path)` не знает что делать при ошибке → `throws IOException`. Вызывающий метод решает: поймать или пробросить дальше.

## Задание 9: NullPointerException — 5 способов получить ★★★
Файл `Task09.java`. Пять примеров NPE (ошибка обращения к null — пустой ссылке): вызов метода на null, доступ к массиву null, распаковка (unboxing) null Integer, цепочка вызовов. Для каждого — как защититься.

## Задание 10: Кофейня — валидация заказа ★★★
Файл `Task10.java`. Проверки: количество < 0? Напиток не в меню? Скидка > 100%? Бросить InvalidOrderException со списком всех нарушений.
