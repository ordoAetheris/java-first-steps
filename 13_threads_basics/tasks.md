# Задания: Многопоточность (Threads — параллельное выполнение)

## Задание 1: Два потока (thread) — параллельный счёт ★☆☆
Файл `Task01.java`. Поток 1 считает 1→50, поток 2 считает 50→1. Запустить оба. Вывод перемешанный — потоки работают параллельно, порядок НЕ гарантирован.
```bash
java Task01
# Thread-1: 1
# Thread-2: 50
# Thread-1: 2     ← порядок не предсказуем!
# Thread-2: 49
```
**Важно:** `t.start()` запускает поток. `t.run()` — НЕ запускает, а выполняет в текущем потоке!

## Задание 2: Параллельное "скачивание" ★☆☆
Файл `Task02.java`. 3 "файла" скачиваются одновременно (`Thread.sleep` имитирует задержку). Каждый — в своём потоке. Итого: ~2 секунды вместо ~6 последовательно.

## Задание 3: Race condition (гонка данных) ★★☆
Файл `Task03.java`. 5 потоков, каждый делает `counter++` 10000 раз. Ожидание: 50000. Реальность: меньше (каждый раз разное число!). Почему? `counter++` — НЕ атомарная (неделимая) операция: read → increment → write. Два потока могут прочитать одно значение.

## Задание 4: synchronized — исправляем гонку ★★☆
Файл `Task04.java`. То же что задание 3, но с `synchronized(lock) { counter++; }`. Теперь результат ВСЕГДА 50000.

## Задание 5: Producer-Consumer (производитель-потребитель) ★★★
Файл `Task05.java`. Один поток кладёт строки в `BlockingQueue` (блокирующая очередь), другой забирает и обрабатывает. `queue.put()` ждёт если очередь полна. `queue.take()` ждёт если пуста.

## Задание 6: Thread.join — дождаться завершения ★★☆
Файл `Task06.java`. 3 потока выполняют вычисления. `t.join()` — подождать пока поток завершится. Собрать результаты ПОСЛЕ завершения всех трёх.

## Задание 7: volatile — видимость между потоками ★★★
Файл `Task07.java`. Поток работает `while(running)`. Другой поток ставит `running = false`. Без `volatile` — первый поток может НЕ увидеть изменение (кеширование процессора)! С `volatile` — видит.

## Задание 8: Deadlock (взаимная блокировка) ★★★
Файл `Task08.java`. Поток 1: lock(A) → lock(B). Поток 2: lock(B) → lock(A). Оба ждут друг друга вечно → программа зависает. `Ctrl+C` для выхода. Как починить? Одинаковый порядок lock'ов.

## Задание 9: ExecutorService — пул потоков ★★☆
Файл `Task09.java`. Вместо `new Thread()` — пул из 4 потоков. `pool.submit(() -> task)` возвращает `Future` (будущий результат). `future.get()` — подождать и получить результат.
```bash
java Task09
# Задача 1: результат 42 (поток pool-1-thread-1)
# Задача 2: результат 17 (поток pool-1-thread-2)
```

## Задание 10: Кофейня — 3 бариста параллельно ★★★
Файл `Task10.java`. `BlockingQueue<Order>` — общая очередь. 3 потока-бариста: `while(true) { Order o = queue.take(); process(o); }`. Основной поток добавляет заказы. Вывод: какой бариста какой заказ обработал.
```bash
java Task10
# Бариста-1: готовит Латте для Павла
# Бариста-3: готовит Эспрессо для Анны
# Бариста-2: готовит Капучино для Бориса
```
