# Java First Steps — от нуля до первого собеседования

## Для кого

Для взрослых людей, решивших сменить профессию. Без снисходительности, без предположения что вы что-то знаете. Но и без воды — только то, что реально используется.

## Подготовка: как работать с файлами на компьютере

Если вы не очень дружите с файловой системой — вот минимум:

### Что такое терминал (командная строка)

Терминал — текстовый интерфейс для управления компьютером. Вместо кликов мышкой — набираете команды.

- **Windows:** нажмите Win+R, введите `cmd` (или найдите "PowerShell" в меню)
- **Mac:** найдите "Terminal" в Applications → Utilities
- **Linux:** обычно Ctrl+Alt+T

### Базовые команды (навигация по файлам)

```bash
pwd                    # показать где я сейчас (print working directory — показать рабочую папку)
ls                     # показать файлы в текущей папке (list — список)
cd first_steps         # перейти в папку first_steps (change directory — сменить папку)
cd ..                  # вернуться на уровень выше
cd ~                   # перейти в домашнюю папку

mkdir my_solutions     # создать папку (make directory — создать папку)
touch Task01.java      # создать пустой файл (Linux/Mac)
```

### Как создать файл с кодом

**Вариант 1: VS Code (рекомендуется)**
1. Установите [VS Code](https://code.visualstudio.com/)
2. Откройте папку: File → Open Folder → выберите `first_steps`
3. Правый клик на папке `01_variables` → New File → `Task01.java`
4. Пишете код, сохраняете (Ctrl+S)

**Вариант 2: из терминала**
```bash
cd first_steps/01_variables
nano Task01.java        # откроется текстовый редактор в терминале
# Пишете код, Ctrl+O для сохранения, Ctrl+X для выхода
```

### Где хранить свои решения

Каждое задание — отдельный `.java` файл **внутри папки темы**:
```
first_steps/
└── 01_variables/
    ├── theory.md          ← прочитать
    ├── tasks.md           ← задания
    ├── hints.md           ← подсказки (если застряли)
    ├── tests/             ← тесты (запускать для проверки)
    ├── Task01.java        ← ← ← ВАШ ФАЙЛ (создаёте сами)
    ├── Task02.java        ← ← ← ВАШ ФАЙЛ
    └── ...
```

**Важно:** имя файла ОБЯЗАНО совпадать с именем класса внутри. Файл `Task01.java` → внутри `public class Task01`.

## Компиляция (перевод кода в машинный формат) и запуск — ТОЛЬКО через терминал

Никакой IDE (среда разработки, например IntelliJ). Открываете текстовый редактор, пишете код, компилируете и запускаете из терминала.

```bash
# Перейти в папку с заданием
cd first_steps/01_variables

# Скомпилировать (превратить ваш текст → байткод, который понимает Java)
javac Task01.java
# Если ошибок нет — появится файл Task01.class
# Если есть ошибки — компилятор покажет номер строки и что не так

# Запустить (JVM выполняет байткод)
java Task01

# Если программа принимает аргументы (входные данные):
java Task09 Павел 28
```

### Два шага: компиляция ≠ запуск

| Шаг | Команда | Что делает | Результат |
|-----|---------|-----------|-----------|
| Компиляция | `javac Task01.java` | Проверяет код, переводит в байткод | Файл `Task01.class` |
| Запуск | `java Task01` | JVM (виртуальная машина) выполняет байткод | Программа работает |

Это РАЗНЫЕ команды. Изменили код → нужно ПЕРЕКОМПИЛИРОВАТЬ (`javac`) → потом ЗАПУСТИТЬ (`java`).

### Почему не IDE

IDE (Integrated Development Environment — интегрированная среда разработки) скрывает от вас что происходит. Вы нажимаете зелёную кнопку и "оно работает". А потом приходите на работу и не понимаете:
- Что компиляция и запуск — это РАЗНЫЕ шаги
- Что `javac` создаёт `.class` файл, а `java` его выполняет
- Как передать параметры в программу
- Зачем нужен classpath (путь, где Java ищет ваши классы)
- Почему "у меня в IDE работает, а на сервере нет"

IDE — потом. Когда поймёте что она делает за вас.

## Порядок тем

```
00_what_is_java      → Что это вообще такое и зачем
01_variables         → Переменные (ячейки для хранения данных) и типы данных
02_execution_control → if/else/switch — ветвление (принятие решений в коде)
03_arrays            → Массивы (наборы элементов)
04_cycles            → Циклы: for, while (повторение действий)
05_methods           → Методы (переиспользуемые блоки кода)
06_oop_basics        → Классы и объекты (основы объектно-ориентированного программирования)
07_oop_advanced      → Наследование, интерфейсы (продвинутое ООП)
08_strings           → Строки (работа с текстом)
09_exceptions        → Обработка ошибок (что делать когда что-то пошло не так)
10_collections       → List, Set, Map (умные контейнеры для данных)
11_io                → Файлы (чтение и запись)
12_generics_enums    → Generics (обобщения) и Enum (перечисления)
13_threads_basics    → Основы многопоточности (параллельное выполнение)
```

## Структура каждой темы

- `theory.md` — кратко: что это, зачем, как выглядит. 5-10 минут чтения.
- `tasks.md` — задания по нарастающей сложности. Последние — с подковыркой.
- `hints.md` — подсказки (направление мысли, не готовое решение).
- `tests/` — готовые тесты. Видите PASS (пройдено) или FAIL (ошибка).

## Проверка заданий

```bash
# Перейти в папку с тестами
cd 01_variables/tests/

# Скомпилировать и запустить тесты
javac RunTests.java && java RunTests

# Результат:
#   PASS: sum = 219            ← правильно
#   FAIL: expected 42, got 0   ← ошибка, нужно исправить
```

Тесты написаны за вас — вы ещё не знаете что такое тестовые фреймворки (библиотеки для тестирования). Просто запускаете и смотрите результат.

## Важное правило

Не нужно запоминать все методы (встроенные функции) и все классы. В Java тысячи классов и десятки тысяч методов. Никто не помнит их все. Нужно знать:

1. **Где искать** — [docs.oracle.com/javase/21/docs/api](https://docs.oracle.com/en/java/javase/21/docs/api/) или просто гуглить "java String methods"
2. **Как читать документацию** — имя метода, параметры, что возвращает, пример
3. **Основные паттерны** — как обойти массив, как найти элемент, как обработать ошибку

Это как словарь при изучении языка: вы не зубрите все слова, вы учитесь быстро находить нужное.
