# Подсказки: Управление выполнением

## Задание 1: Вход в клуб
- `Integer.parseInt(args[0])` — строка → число
- Проверить `args.length > 0` перед доступом

## Задание 2: Оценка
- Цепочка if/else if. Проверяйте от большего к меньшему
- Сначала проверьте некорректные (< 0 или > 100)

## Задание 3: Калькулятор
- `switch(operator)` — String в switch работает с Java 7+
- `*` в терминале = glob, используйте `x` для умножения или кавычки: `java Task03 10 "*" 3`
- Деление: `(double)a / b` для дробного результата

## Задание 4: Скидка
- switch по типу карты
- `String.equals()` для сравнения (НЕ `==`)

## Задание 5: Валидатор
- Цикл по символам: `for (int i = 0; i < password.length(); i++)`
- `password.charAt(i)` — символ на позиции i
- Три boolean флага: hasDigit, hasUpper. Проверять ВСЕ нарушения

## Задание 6: Зоны пульса
- `double percent = (double) pulse / maxPulse * 100;`
- Цепочка if/else if по percent

## Задание 7: Налог
- Считать по частям: Math.min(income, 2_400_000) * 0.13
- Потом: Math.min(Math.max(income - 2_400_000, 0), 2_600_000) * 0.15
- Потом: Math.max(income - 5_000_000, 0) * 0.22

## Задание 8: == для строк
- `==` сравнивает ССЫЛКИ (адреса в памяти)
- `a` и `b` → string pool (один объект) → == true
- `new String()` → новый объект → другой адрес → == false
- ВСЕГДА используйте `.equals()` для строк!

## Задание 9: fall-through
- Без `break` выполнение идёт ДАЛЬШЕ в следующий case
- Иногда это нужно (несколько case → одно действие)
- Обычно — это баг. Всегда ставьте break.

## Задание 10: Кофейня
- Порядок: определить скидку → утро/ночь → ограничить 30% → посчитать итого
- Ночная наценка ВМЕСТО скидки (или поверх? — решить и обосновать)
