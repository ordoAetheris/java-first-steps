## Задание 7: Налоговый калькулятор ★★☆

Файл `Task07.java`. Годовой доход (long из args). Прогрессивная шкала:

| Доход | Ставка |
|-------|--------|
| 0 — 2 400 000 | 13% |
| 2 400 001 — 5 000 000 | 15% |
| Свыше 5 000 000 | 22% |

**Важно:** шкала ПРОГРЕССИВНАЯ. С дохода 6 000 000:
- Первые 2.4M × 13% = 312 000
- Следующие 2.6M × 15% = 390 000
- Оставшиеся 1M × 22% = 220 000
- Итого: 922 000

```bash
java Task07 6000000    # Доход: 6 000 000, Налог: 922 000
```

Допишите комментарий с объяснением. (Подсказка: `==` сравнивает ССЫЛКИ, не содержимое. `equals()` — содержимое.)

## Задание 9: День недели — switch без break ★★★

Файл `Task09.java`. Что выведет?

```java
public class Task09 {
    public static void main(String[] args) {
        int day = 3; // среда

        switch (day) {
            case 1: System.out.println("Понедельник");
            case 2: System.out.println("Вторник");
            case 3: System.out.println("Среда");
            case 4: System.out.println("Четверг");
            case 5: System.out.println("Пятница");
            default: System.out.println("Выходной");
        }
    }
}
```

Запустите. Удивлены? Это **fall-through** — без `break` выполняются ВСЕ case ниже.

Исправьте: добавьте `break` после каждого case. Или перепишите через switch-expression (Java 21).

## Задание 10: Кофейня — скидка + время дня ★★★

Файл `Task10.java`. Аргументы: сумма (double), карта ("gold"/"silver"/"bronze"/"none"), час (int, 0-23).

Правила:
- Скидка по карте (как в задании 4)
- Утренний бонус: с 6:00 до 9:59 — доп. скидка 5% (на уже скидочную сумму)
- Ночная наценка: с 22:00 до 5:59 — наценка 15%
- Если скидка по карте + утренний бонус > 30% — ограничить до 30%

```bash
java Task10 1000 gold 8    # gold(20%) + утро(5%) = 25%. Итого: 750.00
java Task10 1000 gold 23   # gold(20%) + ночь(+15%). Итого: 960.00
java Task10 1000 none 8    # none(0%) + утро(5%). Итого: 950.00
```
