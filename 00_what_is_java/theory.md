# Что такое Java и зачем она нужна

## Программа — это инструкция для компьютера

Компьютер не понимает русский. Он понимает числа — последовательности нулей и единиц. Языки программирования — это способ написать инструкцию на человекочитаемом языке, а потом перевести её в числа.

## Проблема: у каждой операционной системы свои правила

Windows, Linux, macOS — это разные операционные системы. Внутри каждая работает по-своему:
- Файлы хранятся по-разному
- Сетевые вызовы устроены по-разному
- Даже символ переноса строки — разный (`\r\n` в Windows, `\n` в Linux)

Если вы пишете программу на C или C++, вы компилируете её **под конкретную ОС**. Программа для Windows не запустится на Linux. Нужно компилировать заново, менять код под каждую платформу. Это боль.

## Решение Java: "Write once, run anywhere"

Java работает иначе. Между вашим кодом и операционной системой есть прослойка — **JVM** (Java Virtual Machine).

```
Ваш код (.java)
      ↓  javac (компилятор)
Байткод (.class)
      ↓
    JVM  ← вот этот слой адаптируется под каждую ОС
      ↓
 Windows / Linux / macOS
```

Вы пишете код один раз. `javac` компилирует его в **байткод** — промежуточный формат, одинаковый для всех ОС. А JVM — программа, которая установлена на каждой ОС и умеет выполнять этот байткод.

**JVM для Windows** знает как работать с Windows. **JVM для Linux** — с Linux. Ваш код одинаковый. Разница — в JVM.

## Скорость

"Java медленная" — миф из 1990-х. Современная JVM:
- **JIT-компиляция** — JVM наблюдает какой код выполняется часто и компилирует его в машинный код прямо во время работы. После прогрева Java работает на уровне C++ (±10-20%).
- Медленный **старт** (1-3 секунды на загрузку JVM) — но после старта работает быстро.

Для сравнения: Python в 10-100 раз медленнее Java. JavaScript — в 2-5 раз медленнее. Go — примерно на уровне Java. C/C++ — на 10-20% быстрее.

## Где используется Java

### Массовое
- **Банки и финтех** — Тинькофф, Сбер, ВТБ. Почему: надёжность, экосистема, 20 лет production кода.
- **E-commerce** — Ozon, Wildberries (частично). Высокая нагрузка, тысячи запросов в секунду.
- **Микросервисы** — Spring Boot (фреймворк №1 в Java мире). 80%+ Java-вакансий требуют Spring.

### Специализированное
- **Телеком** — протоколы SS7, SMPP, SCCP. Биллинг, маршрутизация звонков. Мало кто лезет — но если залез, ты ценный.
- **WCS (Warehouse Control Systems)** — управление складами. Роботы, конвейеры, сортировка посылок.
- **Big Data** — Hadoop, Spark, Kafka, Elasticsearch. Вся инфраструктура — Java.
- **Android** — мобильные приложения (хотя сейчас Kotlin доминирует, но Kotlin = JVM).

### Почему Java, а не что-то другое?

Честный ответ: часто — потому что её **уже знают**. В компании 50 Java-разработчиков, 10 лет кодовой базы, тонны документации. Переписать на Go или Rust — дорого и рискованно. Java не идеальна, но надёжна и предсказуема.

Плюс:
- Огромное сообщество (вопрос → StackOverflow → ответ уже есть)
- Библиотека на всё (HTTP, базы данных, очереди, crypto, XML, JSON — всё встроено или в одном import)
- Обратная совместимость (код 2005 года компилируется на JDK 21)
- Вакансий больше чем на любом другом языке (в России)

## Как установить Java

```bash
# Ubuntu/Debian
sudo apt install openjdk-21-jdk

# Проверить
java -version   # должно быть 21.x
javac -version  # должно быть 21.x
```

Если у вас Windows — скачайте [Eclipse Temurin JDK 21](https://adoptium.net/) и добавьте в PATH.

## Первая программа

Создайте файл `HelloWorld.java`:

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Привет, мир!");
    }
}
```

Компиляция и запуск:

```bash
javac HelloWorld.java    # создаст HelloWorld.class
java HelloWorld           # выведет: Привет, мир!
```

Что здесь происходит:
- `public class HelloWorld` — объявление класса (контейнера для кода). Имя файла ДОЛЖНО совпадать с именем класса.
- `public static void main(String[] args)` — точка входа. JVM ищет именно этот метод при запуске. Запомните как заклинание — позже поймёте каждое слово.
- `System.out.println(...)` — вывод текста в терминал.

Поздравляю. Вы написали, скомпилировали и запустили Java-программу. Без IDE.
